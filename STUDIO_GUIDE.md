## Как устроен редактор

Редактор состоит из трёх зон:

**Левая панель** — список страниц игры. Здесь создаёшь, переименовываешь,
дублируешь и удаляешь страницы. Страницы можно перетаскивать для сортировки.

**Центральная область** — холст (canvas). Это то, что видит игрок.
Всё что ты размещаешь на холсте — виджеты, кнопки — перетаскивается мышью.
Выбранный элемент показывает рамку и ручки изменения размера.

**Правая панель** — инспектор. Два таба:
- **СТРАНИЦА** — настройки текущей страницы: текст, фон, говорящий, реплики, кнопки, виджеты
- **МИР** — глобальные настройки: переменные, предметы, персонажи, ресурсы

**Шапка (строка над холстом)** — название страницы, её ID, выбор разрешения,
кнопка **+ Виджет** (или клавиша `W`) для открытия библиотеки виджетов.

## Страницы

Страница — базовая единица игры. Каждая сцена, локация, диалог — это страница.

### Что есть у страницы

| Поле | Что это |
|------|---------|
| **Название** | Отображается в списке страниц для тебя (не для игрока) |
| **ID** | Уникальный идентификатор. Используется в действии «Перейти на страницу» |
| **Текст** | Основной текст диалога / нарратива, который видит игрок |
| **Говорящий** | Кто произносит этот текст (Рассказчик, Слэй и т.д.) |
| **Фон** | Фоновое изображение сцены |
| **ПРИ ВХОДЕ** | Действия, которые выполняются автоматически при переходе на страницу |
| **КОНЕЦ ДИАЛОГА** | Действия после последней реплики (см. Диалоговая система) |
| **Кнопки** | Старый способ добавить выбор (сейчас лучше использовать виджет choiceButton) |
| **Виджеты** | UI-элементы поверх фона |

### Переход между страницами

Действие `Перейти на страницу` (goToPage) переносит игрока на другую страницу.
При переходе: индекс диалога сбрасывается, статус «диалог начат» сбрасывается,
оверрайды виджетов сбрасываются. Переменные — **не сбрасываются**, они живут
до конца сессии плейтеста.

## Виджеты

Виджеты — UI-элементы которые ты размещаешь поверх фона. Добавляются через
библиотеку виджетов (`W` или кнопка `+ Виджет` в шапке).

Каждый виджет можно:
- Перетащить мышью
- Изменить размер за угловые / боковые ручки
- Настроить в инспекторе (правая панель → СТРАНИЦА → кликни на виджет)
- Задать условие видимости `visibleWhen`

### Типы виджетов

---

#### 💬 Диалоговое окно (dialogueBox)

Основное окно с текстом. Показывает текст страницы или реплики диалога.

**Настройки:**
- **Источник текста** — `Из страницы` (берёт текст страницы) или `Свой текст` (свой независимый текст)
- **Кликабельно** — если включено, клик по окну переключает реплики. Если ты используешь кнопку `advanceDialogue` — **выключи эту галочку**, иначе игрок сможет кликать мимо кнопки
- **РЕПЛИКИ** — список реплик диалога. Каждая реплика имеет: говорящего, текст, вариант портрета
- **Превью** — кнопки `[текст] [1] [2] [3]` в редакторе показывают как выглядит каждая реплика без запуска плейтеста

**Важно:** если реплик нет — показывается статичный текст страницы.
Если реплики есть — до первого нажатия `advanceDialogue` всё равно показывается
статичный текст. Реплики начинаются только после первого нажатия кнопки.

---

#### 🔘 Кнопки выбора (choiceButton)

Ряд из 1 до 5 кнопок в одной строке. Автоматически центрируются —
если один слот скрыт, остальные перестраиваются.

**Настройки слота:**
- **Текст** — надпись на кнопке
- **Действия** — что происходит при нажатии (можно несколько действий подряд)
- **Видим когда** — условие видимости этого конкретного слота

Количество слотов выбирается кнопками `[1][2][3][4][5]` в инспекторе.

---

#### 🧑 Портрет (portrait)

Изображение говорящего персонажа. Автоматически показывает вариант портрета
(`neutral`, `angry`, `happy` и т.д.) в зависимости от текущей реплики.

---

#### 📝 Текстовый блок (textLabel)

Произвольный текст поверх сцены. Нарратив, подсказки, аннотации.
Поддерживает богатое форматирование (см. Диалоговая система → Теги).

---

#### 🗯️ Речевой пузырь (speechBubble)

Пузырь с указателем-хвостом (сверху, снизу, слева, справа).

---

#### 📊 Полоса значения (intensityBar)

Визуальная шкала переменной. Привязывается к любой числовой переменной
(HP, мана, репутация, накал и т.д.). Цвет меняется по трём порогам.

---

#### ⚡ Быстрое действие (quickAction)

Маленькая иконка-кнопка для быстрых действий. Тип `inventory` открывает
инвентарь. Поддерживает кастомный ассет (изображение).

---

#### 🎒 Инвентарь (inventory)

Кнопка-триггер. При нажатии в плейтесте открывает полное окно инвентаря
с манекеном и слотами экипировки.

---

#### 🖼️ Контейнер (container)

Рамка для группировки элементов. Поддерживает заголовок.

---

### Общие свойства всех виджетов

- **assetId** — привязка UI-ассета (изображения) к виджету вместо стандартного отображения
- **visibleWhen** — условие видимости (см. раздел Условия)
- **actions** — действия при нажатии (на виджет в плейтесте)

## Переменные

Переменные — это **память игры**. Хранят состояние: что игрок сделал,
что произошло, какое значение у параметра.

### Типы переменных

| Тип | Примеры использования |
|-----|----------------------|
| **boolean (да/нет)** | Флаги: `intro_searched`, `quest_started`, `door_opened` |
| **number (число)** | HP, золото, очки репутации, сила, уровень |
| **string (текст)** | Имя персонажа, название выбранного класса |

### Где находятся

Правая панель → таб **МИР** → секция **Переменные**.

Каждая переменная имеет:
- **ID** — внутренний идентификатор (латиница, без пробелов)
- **Название** — отображается в интерфейсе редактора
- **Тип** — boolean / number / string
- **Значение по умолчанию** — начальное значение при старте игры / сбросе

### Как переменные работают

Переменная сама по себе ничего не делает. Она становится полезной когда:

1. **Действие** её меняет (`setVariable`, `addToVariable` и т.д.)
2. **Условие** (`visibleWhen`) на неё смотрит

**Пример:** Кнопка "Искать" должна исчезнуть после поиска:
- Переменная `intro_searched` = `false` по умолчанию
- На слоте "Искать": `visibleWhen` → `intro_searched == Нет`
- В `afterCollect` награды: `setVariable(intro_searched, true)`
- Когда игрок получает предметы → переменная меняется → условие перестаёт выполняться → кнопка исчезает

## Предметы и инвентарь

### Создание предметов

Правая панель → **МИР** → **Предметы** → `+ Предмет`.

Каждый предмет:
- **ID** — уникальный идентификатор
- **Название** (ru/en)
- **Тип** — оружие, броня, расходуемый, ключевой и т.д.
- **Редкость** — обычный, необычный, редкий, эпический, легендарный
- **Иконка** — путь к изображению
- **Слот экипировки** — куда надевается
- **Модификаторы** — бонусы к переменным при ношении
- **Урон оружия** — добавляется к итоговому урону
- **Прочность** — опционально
- **quantityVariableId** — привязка количества предмета к переменной

### Инвентарь игрока

Открывается действием `openInventory` или виджетом 🎒.

**Что умеет:**
- Показывает рюкзак (размер = значение Выносливости)
- Показывает манекен со слотами экипировки
- Drag & drop предметов из рюкзака на манекен
- Фильтры по типу и редкости
- Контекстное меню: Одеть / Посмотреть / Выбросить
- При экипировке показывает дельту характеристик (до/после)

### Модалка награды (showItemReward)

Открывается действием `showItemReward`. Показывает предметы с иконками
и кнопкой «Получить».

**После нажатия «Получить»:**
1. Предметы попадают в инвентарь
2. Выполняются действия из поля **«При нажатии «Получить»»** (afterCollect)

Именно в `afterCollect` нужно ставить `setVariable` для скрытия кнопок,
открытия новых путей и т.д.

## Действия (Actions)

Действие — это то, что происходит в ответ на нажатие кнопки или другое событие.
Одна кнопка может выполнять несколько действий подряд.

### Навигация

| Действие | Что делает |
|----------|-----------|
| `Перейти на страницу` | Переносит игрока на другую страницу |
| `Начать квест` | Запускает квест по ID (система в разработке) |

### Переменные

| Действие | Что делает |
|----------|-----------|
| `Установить переменную` | Ставит конкретное значение (`true`, `false`, число) |
| `Добавить к переменной` | Прибавляет число (для числовых переменных) |
| `Вычесть из переменной` | Вычитает число |

### Предметы

| Действие | Что делает |
|----------|-----------|
| `Выдать предмет` | Добавляет предмет в инвентарь игрока |
| `Забрать предмет` | Убирает предмет из инвентаря |

### Интерфейс

| Действие | Что делает |
|----------|-----------|
| `Открыть инвентарь` | Открывает модальное окно инвентаря |
| `Показать награду` | Показывает модалку с предметами + кнопка «Получить» |
| `Продвинуть диалог` | Запускает/листает реплики диалога (1й клик = старт, следующие = следующая реплика, после последней = onDialogueEnd) |

### Диалог и портреты

| Действие | Что делает |
|----------|-----------|
| `Сменить вариант портрета` | Меняет выражение портрета (angry, happy, sad...) |
| `Установить накал` | Меняет значение переменной накала (для intensityBar) |
| `Изменить свойство виджета` | Меняет любое свойство виджета во время игры |

### Ресурсы и отношения

| Действие | Что делает |
|----------|-----------|
| `Выдать ресурс` | Добавляет монеты / бензин / кристаллы |
| `Изменить отношения` | Меняет отношение персонажа к игроку |
| `Изменить репутацию` | Меняет репутацию |
| `Изменить стат игрока` | Меняет силу, уровень и т.д. |

### Где можно ставить действия

- На **слот кнопки выбора** (choiceButton)
- В **ПРИ ВХОДЕ** страницы (выполняется при переходе на страницу)
- В **КОНЕЦ ДИАЛОГА** страницы (выполняется после последней реплики)
- В **«При нажатии «Получить»»** модалки награды (afterCollect)

## Условия — видим когда

Условие `visibleWhen` — это фильтр видимости. Элемент показывается
**только если условие выполняется**.

Работает на:
- Виджетах целиком
- Отдельных слотах кнопок выбора
- Кнопках (старый формат)

### Типы условий

| Тип | Пример |
|-----|--------|
| **Переменная** | `intro_searched == Нет` |
| **Количество предмета** | `sword_qty >= 1` |
| **Отношения** | `relationship_mila >= 50` |
| **Репутация** | `reputation > 20` |
| **И / ИЛИ / НЕ** | Комбинации нескольких условий |

### Главный принцип (запомни!)

Чтобы что-то **исчезло после события**:

```
1. Создай переменную (например: door_opened = false)
2. Поставь visibleWhen на элемент: door_opened == false
3. Добавь действие setVariable(door_opened, true) в нужный момент
```

Без `visibleWhen` переменная ничего не скрывает — она просто хранит значение.

## Диалоговая система

### Статичный текст vs Реплики

**Статичный текст** — поле «Текст» на странице. Это нарратив или описание
которое игрок видит сразу. Говорящий — из поля «Говорящий» страницы.

**Реплики (dialogueLines)** — очередь реплик которые листаются последовательно.
Каждая реплика может иметь своего говорящего и вариант портрета.

**Ключевой момент:** статичный текст и реплики не конфликтуют.
- До нажатия `advanceDialogue` — показывается статичный текст
- После первого нажатия — начинаются реплики

### Как настроить систему «кнопка листает реплики»

1. На странице добавь **текст** (то что игрок видит изначально)
2. Добавь **реплики** в инспекторе dialogueBox (кнопка `+ Добавить`)
3. На слоте кнопки (например «Искать») поставь действие `Продвинуть диалог`
4. В настройках dialogueBox **сними галочку «Кликабельно»**
5. В секции **КОНЕЦ ДИАЛОГА** добавь что происходит после последней реплики

### Конец диалога (onDialogueEnd)

Срабатывает когда игрок нажимает `advanceDialogue` после того как показана
последняя реплика. Типичные действия:
- `showItemReward` — показать награду
- `setVariable` — поменять флаг
- `goToPage` — перейти на следующую сцену

### Теги форматирования в тексте

В любом тексте диалога работают теги:

| Тег | Эффект |
|-----|--------|
| `[b]текст[/b]` | **Жирный** |
| `[i]текст[/i]` | *Курсив* |
| `[color=#FF0000]текст[/color]` | Цветной текст |
| `[size=20]текст[/size]` | Размер шрифта |
| `[pause]` | Пауза в typewriter-анимации |
| `[shake]` | Эффект тряски текста |

### Typewriter-анимация

Текст печатается посимвольно с анимацией. Скорость:
- Стандартная — 3 символа за кадр
- На многоточии `...` — медленнее
- При высоком накале (intensityBar > 70) — быстрее

## Плейтест

### Запуск и кнопки

- **Плейтест** — кнопка в левой панели. Запускает тест с текущего состояния.
- **Загрузить** — восстанавливает последнее сохранённое состояние
- **Сброс** — начать с нуля, как будто игра запущена впервые
- **Выйти в редактор** — вернуться к редактированию

### Автосохранение

При нажатии кнопок (в процессе плейтеста) прогресс сохраняется автоматически.
При повторном входе в плейтест — сохранение подгружается.

Это значит: если ты перешёл на страницу 2 и вышел из плейтеста,
при следующем входе ты окажешься на странице 2 с теми же переменными.

Чтобы начать с нуля — нажми **Сброс**.

### Разрешение холста

Выбирается в шапке:
- `960×600` — устаревший формат 16:10
- `1280×720` — стандарт 16:9 HD (рекомендуется)
- `1600×900` — 16:9
- `1920×1080` — Full HD

Все позиции элементов хранятся в процентах, поэтому при смене
разрешения всё масштабируется автоматически.

---

## Главное правило редактора

> **Переменная = память. Условие = реакция.**

Эти два понятия работают только в паре:

```
СОБЫТИЕ → действие setVariable → переменная меняется
                                         ↓
                              visibleWhen проверяет переменную
                                         ↓
                              элемент показывается или скрывается
```

Если что-то не скрывается — скорее всего либо нет `visibleWhen`,
либо условие написано неправильно, либо `setVariable` не стоит в нужном месте.

## Что планируется добавить

Список функций которые имеют смысл для проекта и логично вписываются
в существующую архитектуру. Расставлены по приоритету.

---

### Высокий приоритет

**Система квестов**
Действие `startQuest` уже существует в коде, но не реализовано полностью.
Нужен: список квестов, условия завершения, отображение в UI.

**Переходы между страницами**
Анимации при переходе: fade, slide, wipe. Сейчас переход мгновенный.
Настраивается на действии `goToPage`.

**Звуковая система**
Фоновая музыка на странице + звуки событий (при нажатии кнопки, при получении предмета).
Действия: `playMusic`, `playSound`, `stopMusic`.

**Экспорт игры**
Возможность собрать игру в отдельный HTML-файл или архив,
чтобы поделиться с игроками без Studio.

---

### Средний приоритет

**Карта мира**
Виджет карты с точками-локациями. Переход на страницу кликом по точке.
Точки могут быть скрыты по условиям (visibleWhen).

**Система магазина**
Страница-шаблон: список товаров с ценами, кнопка «Купить»,
проверка наличия монет перед покупкой.

**Несколько слотов сохранения**
Сейчас один слот автосохранения. Можно добавить 3–5 ручных слотов.

**Кастомный HUD (ресурсная панель)**
Сейчас верхняя полоса с монетами/бензином/кристаллами фиксированная.
Нужна возможность настраивать что там показывается.

**Уведомления во время игры**
Всплывающие сообщения: «Получено: Меч +1», «Квест обновлён».
Срабатывают от действий.

### Низкий приоритет / будущее

**Система крафта**
Рецепты: предмет A + предмет B = предмет C. Таблица рецептов,
кнопка «Скрафтить», проверка наличия ингредиентов.

**Система времени / циклы**
Переменная «время дня» (утро/день/вечер/ночь), автоматически меняется
при переходах. Фоны и тексты зависят от времени.

**Рандомизация**
Действие `rollDice` — случайное число в диапазоне → в переменную.
Для боевых сцен, случайных событий.

**Боевая система**
Пошаговый бой: кнопки «Атака / Защита / Бежать», расчёт урона
на основе переменных (сила, оружие, броня).

**Локализация EN**
Структура данных уже поддерживает `ru` и `en` поля везде.
Нужен UI переключатель языка в плейтесте.

**Дерево истории (Story Graph)**
Визуальное дерево всех страниц с переходами между ними.
Видно «куда ведёт» каждая страница.
