# Narrative Systems — Plan

Цель: инструментарий для нелинейного повествования в Studio.
Без этих блоков сценарист вынужден дублировать страницы для каждого состояния сцены — не масштабируется.

---

## Блок 1 — `onEnter` на странице ✅ в работе

**Проблема:** при навигации на страницу нет способа автоматически выполнить действия (поставить флаг, выдать предметы). Всё требует явного клика игрока.

**Решение:** `onEnter?: ButtonAction[]` на `StudioPage`.  
При `executeAction → goToPage` после смены страницы выполнять все `onEnter` actions последовательно.

**UI (PageSection):** секция "ПРИ ВХОДЕ" — список действий, каждое через `ActionEditor`, кнопка "+ Добавить действие", удаление крестиком.

**Даёт сценаристу:**
- Автоматически выдать предметы при входе на сцену
- Поставить флаг `visited = true` без кнопки
- Изменить переменную, запустить квест, сменить интенсивность — при переходе

---

## Блок 2 — visibleWhen редактор для виджетов

**Проблема:** `visibleWhen` у виджетов технически работает в playtest (evaluateCondition против живых variableValues), но задать его через UI невозможно.

**Решение:** в инспекторе виджета секция "ВИДИМОСТЬ" с `ConditionEditor` (компонент уже есть), привязанная к `w.visibleWhen`.

**Даёт сценаристу:**
- Показывать/скрывать выборы на основе флагов и переменных
- Choice 1 исчезает после посещения сцены, появляется Choice 2
- Условные элементы UI (подсказки, портреты, кнопки) без дублирования страниц

---

## Блок 3 — Playtest Save / Reset

**Проблема:** `playtestState.variableValues` живёт только в RAM. Обновление браузера сбрасывает весь прогресс. Тестировать ветки невозможно без прохождения с нуля.

**Решение:**
- Автосохранение: `localStorage.setItem('slay-playtest-save', ...)` при каждом `executeAction`
- При старте playtest: загружать сохранение если есть, иначе стартовые значения
- Playtest-бар: индикатор "● Сохранено" + кнопка "↺ Сбросить прогресс"

**Даёт сценаристу:**
- Тестировать конкретную ветку, не проходя заново от начала
- Сбросить и начать с чистого листа одной кнопкой
- Прогресс выживает при обновлении страницы

---

## Блок 4 — Multi-action (список действий на кнопке) ✅ готово

**Проблема:** одна кнопка = одно действие. Нельзя "выдать предмет + поставить флаг + перейти на страницу" за один клик.

**Решение:** `actions?: ButtonAction[]` на `UIWidget`.  
При playtest: если `actions` есть — выполнять по очереди. `action` (singular) остаётся как запасной для backward compat.

**UI:** в инспекторе виджета список с "+ Добавить действие" и удалением каждого пункта.

**Даёт сценаристу:**
- Кнопка выбора, которая одновременно даёт предмет, меняет переменную и переходит на страницу
- Основа для RPG-событий, боёвки, торговли — любого действия с несколькими последствиями

---

## Схема мини-квеста (целевое состояние после блоков 1–3)

```
Страница 1:
  dialogueBox: реплики нарратора
  Choice 1 → page2    visibleWhen: visited_p2 == false
  Choice 2 → page3    visibleWhen: visited_p2 == true

Страница 2:
  onEnter: [giveItem(меч,1), giveItem(щит,1), setVariable(visited_p2, true)]
  dialogueBox: несколько реплик нарратора
  "Вернуться" choiceButton → page1

Страница 3:
  следующая сцена
```

Переменная `visited_p2` — тип boolean, default false — создаётся автором в Variables.

---

## Зависимости и порядок

```
Блок 1 (onEnter)        → разблокирует основной сценарий
Блок 2 (visibleWhen UI) → требует наличия переменных из Блока 1
Блок 3 (Save/Reset)     → независим, делает тестирование вменяемым
Блок 4 (Multi-action)   → независим, нужен для RPG-событий позже
```

Блоки 1–3 достаточны для полного прохождения мини-квеста без дублирования страниц.
